Change the priority of a thread.

The thread will be entered at the end of the ready queue at the corresponding priority. The new priority setting will be valid until the thread is terminated or the priority is changed again.

Parameters
tidThread ID
priorityNew priority (1-127)
Returns

◆ CreateSema()

int CreateSema ( SemaParam *  sparam)

Create a counting semaphore object.

Up to 256 semaphores can be created, 3 of which are reserved during runtime initialization by crt0.s (2 from _InitSys and 1 from InitThread).

Parameters
sparamInitial semaphore parameters
Returns
Semaphore ID on success, -1 if creation would exceed the maximum number of semaphores or a negative value was passed for sema->initCount

Release semaphore resource.

If the sid value is 0 and there is no free space in the semaphore queue, the thread at the wait queue start will be released and placed in READY state. In all other cases, the value of the semaphore is incremented.
